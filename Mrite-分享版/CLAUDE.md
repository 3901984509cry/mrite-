# 数学建模竞赛求解+论文自动生成模板

## ⛔ 全自动模式（最高优先级）

**本模板设计为全自动无人值守运行。以下规则优先级高于一切：**

1. **禁止任何形式的用户交互**：不询问、不确认、不等候、不提示选择
2. **禁止中断等待**：遇到任何决策点自行判断并继续，不做 `ask` 类操作
3. **从头到尾一次性完成**：读取输入 → 生成计划 → 逐题求解 → 撰写论文 → 编译 PDF，中间不停
4. **编译后自动排版优化**：无需询问，直接执行
5. **即使遇到不确定性也自行选择最优方案继续**，不做任何停顿

```
│   ├── format.cls
│   ├── fonts/
│   ├── 0.摘要.tex ~ 4.符号说明.tex
│   ├── 5.模型的建立与求解.tex
│   ├── 5.1.问题1的建立求解.tex    ← 主文件，\input 两个子文件
```

## 一、模式选择

本模板支持两种执行模式：

### 模式A：论文格式提取模式

**触发**：用户说"提取论文格式"/"学习这篇论文的写法"/"提取这个模板"/"根据这篇论文做模板"。

**目的**：将给定的参考论文转换为可复用的 `论文/` 模板文件夹，用于后续竞赛论文生成。

**输入要求**：
- 参考论文放在 `题目/` 文件夹（PDF或DOCX格式）
- 可选：数据文件放在 `数据/` 文件夹（用于理解数据引用格式）

**执行流程**：见 [二、论文格式提取流程](#二论文格式提取流程)

### 模式B：竞赛题目求解模式

**触发**：用户说"开始求解"/"求解这个题目"/"按规则求解"。

**目的**：自动求解竞赛题目并生成完整论文。

**输入要求**：
- 赛题文件放在 `题目/` 文件夹（PDF或DOCX格式）
- 附件数据放在 `数据/` 文件夹
- `论文/` 文件夹已存在（模板）

**执行流程**：见 [三、竞赛题目求解流程](#三竞赛题目求解流程)

## 二、论文格式提取流程

**全自动模式：全程无需用户确认，不询问、不等待、不中断。从读取论文到生成模板一气呵成。**

### Step 0：读取参考论文

1. 定位 `题目/` 文件夹下的论文文件（优先PDF，其次DOCX）
2. 提取全文内容：
   - PDF：使用 PyPDF2 逐页提取文字
   - DOCX：使用 python-docx 提取段落和样式
3. 保存原文到临时位置（后续步骤会移到 `论文_新模板/参考论文原文.txt`）

### Step 1：结构分析

分析论文结构并生成结构分析报告（后续写入 `论文_新模板/论文结构分析.md`），包含：

**1.1 章节层次提取**
- 识别一级标题（section）、二级标题（subsection）、三级标题（subsubsection）
- 记录每个章节的编号、标题、起始位置
- 构建章节树状结构

**1.2 格式特征识别**
- **封面信息**：标题位置、作者信息、单位信息、日期格式、特殊标识（如赛题编号）
- **摘要格式**：摘要位置（封面页/独立页）、关键词格式、摘要长度特征
- **目录格式**：是否包含目录、目录深度（到几级标题）
- **正文格式**：段落缩进、行距、字体大小、页边距特征
- **图表格式**：
  - 图片引用格式（`\includegraphics` 参数、宽度设置）
  - 图片标题位置（上方/下方）、编号格式
  - 表格样式（三线表/全框线表）、列宽设置方式
  - 表格标题位置（上方/下方）
- **公式格式**：编号位置（右对齐/左对齐）、编号格式
- **参考文献格式**：管理方式（bibtex/纯tex）、引用样式（数字/作者-年份）、条目格式
- **页眉页脚**：是否有页眉、页码位置、特殊标识

**1.3 章节内容模式识别**

对每个章节分析其**组织模式**：

| 章节类型 | 典型模式 | 模板化策略 |
|---------|---------|----------|
| 摘要 | 开头段+逐问描述+结尾段+关键词 | 提取"逐问描述"的句式模板 |
| 引言 | 背景+意义+问题重述 | 识别各部分的段落数和过渡词 |
| 文献综述 | 分类综述+研究现状+本文创新 | 识别分类维度和文献引用密度 |
| 方法/模型 | 问题分析+模型建立+求解方法 | 识别公式密度、步骤划分方式 |
| 结果 | 表格+图片+文字分析 | 统计图表数量、分析段落结构 |
| 讨论 | 结果解释+对比分析+局限性 | 识别讨论维度和段落组织 |
| 结论 | 工作总结+主要发现+展望 | 提取总结句式和展望模板 |
| 参考文献 | 文献列表 | 提取格式规范和条目数量 |
| 附录 | 代码/数据/补充材料 | 识别附录组织方式 |

**1.4 特殊元素识别**
- 流程图/框图：是否使用、绘制工具（tikz/外部图片）
- 算法伪代码：是否包含、使用的环境（algorithm/algorithmic）
- 数学符号表：是否独立成章、表格格式
- 缩略语表：是否包含
- 致谢：位置（参考文献前/后）

### Step 2：生成新模板文件夹

在项目根目录创建 `论文_新模板/` 文件夹，包含以下内容。

**⚠️ 替换对齐原则（最重要）**

新模板必须能够直接 `mv 论文_新模板 论文` 替换原模板，并且后续"竞赛求解模式"完全无需修改流程即可使用。因此：

1. **必含章节骨架**：无论参考论文章节如何命名，新模板必须包含以下文件，文件名一字不差：
   ```
   论文.tex            ← 主文件，按下文规则改写
   format.cls          ← 直接复制原项目 论文/format.cls
   fonts/              ← 直接复制原项目 论文/fonts/
   0.摘要.tex
   1.引言.tex
   2.总体分析.tex
   3.模型假设.tex
   4.符号说明.tex
   5.模型的建立与求解.tex
   5.1.问题1的建立求解.tex
   5.1.1.分析与准备.tex
   5.1.2.建模与求解.tex
   6.模型检验.tex
   7.模型评价.tex
   8.模型改进推广.tex
   9.参考文献.tex
   10.附录.tex
   ```
   这是因为求解流程的 `5.模型的建立与求解.tex`、`5.1.X` 复制改名机制依赖此命名。

2. **风格注入到内容，结构保持不变**：参考论文的格式特征（封面布局、字体设置、宏包选择、表格样式、图片宽度、参考文献格式、章节内的段落组织模式等）注入到上述固定文件的**内容**和 `format.cls`/`论文.tex` 导言区里，而**不是**改变文件骨架。

3. **章节内容映射规则**：把参考论文的章节内容**映射**到上述10个章节，而非新建章节文件：

   | 参考论文章节（无论叫什么） | 映射目标文件 |
   |---|---|
   | 摘要 / Abstract / 内容提要 | `0.摘要.tex` |
   | 引言 / 研究背景 / Introduction / 问题重述 | `1.引言.tex` |
   | 总体分析 / 问题分析 / 总体思路 | `2.总体分析.tex` |
   | 模型假设 / 基本假设 / Assumptions | `3.模型假设.tex` |
   | 符号说明 / 变量说明 / 记号约定 | `4.符号说明.tex` |
   | 模型建立 / 方法 / 模型与求解 / 主体章节 | `5.模型的建立与求解.tex` 及 `5.1.X` 子文件 |
   | 模型检验 / 误差分析 / 灵敏度分析 / 验证 | `6.模型检验.tex` |
   | 模型评价 / 优缺点 / 讨论 | `7.模型评价.tex` |
   | 模型改进 / 推广 / 展望 | `8.模型改进推广.tex` |
   | 参考文献 / References | `9.参考文献.tex` |
   | 附录 / 附件 / 代码清单 | `10.附录.tex` |
   
   参考论文中没有对应章节的，目标文件保留为占位符注释；参考论文中多出来的章节，并入语义最接近的目标文件。

4. **图片路径统一**：所有 `\includegraphics` 路径统一改写为 `../求解/问题X/图片/xxx.png`，不沿用参考论文的原始路径。

5. **辅助文档落位**：分析报告 `论文结构分析.md`、参考论文原文 `参考论文原文.txt`、模板说明 `README.md` 全部放在 `论文_新模板/` 内部，不污染 `求解/`。

**2.1 主文件生成：`论文.tex`**

```latex
% 根据参考论文提取的导言区内容
\documentclass[选项]{文档类}
\usepackage{...} % 保留参考论文使用的宏包

% 根据参考论文的封面格式生成封面代码
\title{论文标题}
\author{作者信息}
% ...

\begin{document}

% 封面
% ...

% 按参考论文的章节顺序生成 \input{} 语句
\input{0.摘要.tex}
\input{1.引言.tex}
% ...
\input{N.结论.tex}
\input{参考文献.tex}
\input{附录.tex}

\end{document}
```

**关键提取规则**：
- 完整复制 `\documentclass` 行（包含所有选项）
- 保留所有 `\usepackage` 行（按原顺序）
- 保留所有 `\newcommand`、`\renewcommand`、`\newenvironment` 定义
- 提取页面设置（`\geometry`、`\setlength` 等）
- 提取字体设置（`\setmainfont`、`\setCJKmainfont` 等）

**2.2 章节文件生成**

按"替换对齐原则"中固定的10个目标文件名生成，**不**新建自由命名文件。每个章节文件的内容遵循：

- **章节命令固定**：第一行为 `\section{原模板对应的中文标题}`（如 `\section{引言}`、`\section{符号说明}` 等），不使用参考论文的英文/异名标题。
- **子节结构来自参考论文**：参考论文该章节内的 `\subsection{}` / `\subsubsection{}` 划分方式被保留下来，子节标题翻译/规范化为中文。
- **段落组织模式来自参考论文**：把参考论文该章节的"段落数量、每段功能、过渡词、句式"提炼为带注释的占位符，写在文件里。
- **风格细节来自参考论文**：表格样式、图片宽度、公式编号方式、列表与否等沿用参考论文的写法。

特别地：

- `5.模型的建立与求解.tex` 仍只写一个 `\section{模型的建立与求解}` + 按问题数 `\input{5.X.问题X的建立求解.tex}`，**不**直接铺正文。问题数 N 在求解阶段才确定，因此这里只放 `\input{5.1...}` 一行示例 + 注释提示如何按 N 增减。
- `5.1.问题1的建立求解.tex` 内部 `\input{5.1.1.分析与准备.tex}` 和 `\input{5.1.2.建模与求解.tex}`。子节内的段落骨架（具体分析→流程图→模型准备；模型建立→模型求解）按参考论文该类章节的组织模式重写。

每个章节文件的内容由两部分构成：
- **保留章节命令**：`\section{标题}`（中文，与原模板一致）
- **提取内容模板**：将具体内容替换为**带注释的占位符**

**占位符生成规则**：

```latex
% 原文：
\section{引言}
\subsection{研究背景}
随着人工智能技术的快速发展，深度学习在各个领域取得了显著成果。
本文针对XXX问题，提出了基于YYY的解决方案。

% 生成模板：
\section{引言}
\subsection{研究背景}

% 第1段：宏观背景（2-3句话，从领域发展趋势/社会需求引入）
% 示例：随着人工智能技术的快速发展，深度学习在各个领域取得了显著成果。

% 第2段：问题来源（1-2句话，说明本文要解决的具体问题）
% 示例：本文针对XXX问题，提出了基于YYY的解决方案。
```

**2.3 文档类文件处理：`format.cls`（如有）**

- 如果参考论文使用自定义文档类（如 `\documentclass{cumcmthesis}`），尝试：
  1. 在 `论文.tex` 导言区查找是否有 `format.cls` 或类似文件的引用

### Step 4：编译测试

1. 在 `论文_新模板/` 文件夹内编译测试：
```bash
cd 论文_新模板
xelatex -interaction=nonstopmode 论文.tex
xelatex -interaction=nonstopmode 论文.tex
```

2. 检查编译结果：
   - 0 Error（必须）
   - Warning 数量（记录到 README.md）
   - 生成的 PDF 页数、目录结构

3. 如有 Error：
   - 分析错误原因（缺失宏包/字体/语法错误）
   - 自动修复（补充宏包、调整语法）
   - 重新编译直到 0 Error

4. 编译成功后，在 `README.md` 末尾追加：
```markdown
## 编译测试结果

- Error 数量：0
- Warning 数量：X
- 生成 PDF：论文.pdf（共 X 页）
- 测试时间：YYYY-MM-DD HH:MM:SS
```

## 三、竞赛题目求解流程

**触发**：用户说"开始求解"/"求解这个题目"/"按规则求解"。

**全自动模式：全程无需用户确认，不询问、不等待、不中断。从读取输入到论文编译完成一气呵成。**

### Step 0（求解模式）：读取输入
1. 读 `题目/`（PDF用PyPDF2，DOCX用python-docx）
2. 读 `数据/` 下所有xlsx/docx，**全量读取**，不只前几行。仔细检查：
   - 每个sheet的列数是否每行一致
   - 每列数据类型是否统一（数值列里有没有混文字）
   - 是否有合并单元组、多级表头、空行分隔、底部备注行
   - 是否有隐藏sheet
3. 打印数据总览：每个sheet的 行数×列数、列名、数据类型、异常行（如有）
4. 自动识别问题数量N

### Step 1：求解计划
读题后，写 `求解/求解计划.md`，写完直接开干。六章结构：
- **题目总体方向**（一整段）：问题类型、核心任务、数据规模、研究递进逻辑
- **各题求解思路**（每问含具体分析+建模+方法匹配表）：文章风格，不分点。具体分析写输入/输出/难点/定位，建模写核心公式+方法选择理由，末尾附表列出主方法、备选方法、关键算法/工具
- **各题输出标准**（含图表规划+预期输出）：每问必须产出的图表、表格、数值结果清单
- **操作步骤**（含依赖顺序）：每问从加载→预处理→建模→求解→输出的完整链路，标注数据在问题间的流转
- **文件清单**：代码/图/CSV文件名（中文）
- **异常预案**：数据缺失、模型不收敛等备选处理

### Step 2：逐题求解
1. 按N创建 `求解/问题一/图片/` ~ `求解/问题N/图片/`（如不存在），创建 `求解/问题一/结果/` ~ `求解/问题N/结果/`（如不存在）
2. 每个问题一个py文件，中文命名，放在 `求解/问题<序号>/`
3. py文件内部**先算后画**：先完成所有计算得到数值结果，打印每组数据的 min/max/mean/std/CV/amplitude（供论文引用），然后画图（每题至少 4-6 张图，覆盖主要分析维度，多多益善）
4. 问题1输出 `求解/预处理数据.csv` 供后续共用
5. 如有 `Glyph missing from font` 等中文渲染警告，修正字体配置后重跑；如脚本报错，修复后重跑，通过后再下一问
6. **灵敏度分析前置**：在求解阶段就测试关键参数（如树的数量、学习率、正则化系数）对结果的影响，记录最佳参数组合，供论文第6章直接引用

### Step 3：写论文
按第二章规范逐章写tex。
- `5.模型的建立与求解.tex` 按N生成 `\input{5.X}`
- 每个 `5.X.问题X的建立求解.tex` 内部拆为2个子文件：`5.X.1.分析与准备.tex`（具体分析+流程图+模型准备）和 `5.X.2.建模与求解.tex`（模型建立+模型求解）。从 `5.1.1~5.1.2` 模板复制改名
- 拆分原则：模型建立和模型求解必须在一起（求解大量引用建立的公式/变量/参数），拆开适得其反

### Step 4：编译
```bash
cd 论文
xelatex -interaction=nonstopmode 论文.tex
xelatex -interaction=nonstopmode 论文.tex
```
`grep -c 'Error' 论文.log` 必须为0。有 Error 则修复重编译直到 0。

### Step 5（自动）：排版优化

编译0错误后，自动执行排版优化直到所有问题清除：
1. `grep 'Overfull\|Underfull\|Float too large' 论文.log`
2. 根据 warning 定位对应 tex 文件的具体行
3. 修复（缩短表格文字、调整图表尺寸、收紧段落等）
4. `xelatex ×2` 重编译，重新检查
5. 直到日志干净、排版紧密、无溢出无大面积空白、图表正常

**全程无需用户确认，自动执行。**

## 二、论文规范

### 通用规则

- **模板为默认结构，如与题目实际内容不适配，AI自行调整**，不硬套
- tex文件中只写可编译的LaTeX代码，不可出现 `...（提示语）` 等模板指令
- 正文一律自然段落，禁止分点符号（1. 2. 3. 或 ● ○ 等）
- 禁止正文加粗（`\textbf` 仅用于摘要中的"对于问题X"和问题重述中的"问题N："）
- 图宽 `0.8\textwidth`，表宽 `\textwidth`
- 所有图表必须被引用（"如图X所示""如表X所示"），图表标题/caption/坐标轴全中文
- 全文任何位置（章与章、节与节、子节与子节之间）一律不加 `\newpage`，内容自然接续，禁止人为断页
- 图文件不画 `set_title()`，标题由 `\caption{}` 承担
- **表格格子文字一行显示，不换行**

### 各章节详细规范

#### 0. 摘要

**字数硬约束**：

| 段落 | 字数 | 说明 |
|------|------|------|
| 开头段 | ≤30字 | 固定 |
| 结尾段 | ≤20字 | 固定 |
| **总计** | **≤900字** | **不可超过** |
| 页数 | 必须=1页 | 编译后验证 |

**各问题字数按类型弹性分配**（总计扣除头尾后≈850字在各问题间分配）：

| 问题类型 | 摘要占比 | 说明 |
|----------|----------|------|
| 建模求解型（优化/预测/分类/评价） | 高（≈30-35%） | 方法+求解+结果占篇幅 |
| 数据分析型（统计/探索/可视化） | 中（≈20-25%） | 处理+发现+结论 |
| 建议总结型（建议/总结/改进） | 低（≈10-15%） | 一笔带过即可 |

例：4问含2个建模型+1个分析型+1个建议型 → 建模型各≈250字、分析型≈180字、建议型≈120字、头尾50字=850字。 |

**直接复制以下模板框架，每问选类型替换 `【】`：**

```latex
\begin{abstract}
本文针对【题目背景】问题，提出了基于【主要方法】的系统解决方案。

【每问从这里复制，选对应类型模板填入】

综上，本文为【领域】问题提供了高效解决方案。

\keywords{【关键词1】；【关键词2】；【关键词3】；【关键词4】；【关键词5】；【关键词6】}
\label{abstract:end}
\end{abstract}
```

**每问三种类型，对号入座**：

**① 有数据**（题目提供附件数据，含预处理→建模→求解全流程，预测/分类/优化/评价等均归此类）：
```
\textbf{针对问题X：}本问是【类型】问题，需【要求】。首先，对附件数据预处理，提取【特征】，【清洗/标准化等操作】。接着，【变量筛选/降维/参数选择】。然后，采取【模型】进行【求解过程】。最后，【对比/验证/排序】，给出【输出】。结果讨论显示，\textbf{【数值结果】}，模型有效地【评价】。
```

**② 无数据**（无附件数据，直接建立机理/方程/优化模型，仿真/推导/理论分析等均归此类）：
```
\textbf{针对问题X：}本问是【类型】问题，需【要求】。首先，分析【机理/物理过程/目标约束】，建立【方程/动力学/优化模型】。接着，设定【参数/初始条件】。然后，通过【数值方法/仿真工具/求解算法】进行求解，得到【结果】。结果讨论显示，\textbf{【数值结果】}，模型有效地【评价】。
```

**③ 建议型**（基于前文结果给出建议/策略/总结，一笔带过）：
```
\textbf{针对问题X：}本问是建议总结问题，需【要求】。基于前文分析，从【角度1】和【角度2】出发，给出【建议数量】条建议。核心结论为\textbf{【要点】}。
```

**一页验证（编译后必做）**：

```bash
# 编译论文后，检查摘要是否超出一页
PAGE=$(grep 'abstract:end' 论文/论文.aux | grep -oE '\{[0-9]+\}' | head -1 | tr -d '{}')
if [ "$PAGE" -gt 1 ]; then
  echo "摘要超出一页（第${PAGE}页），需缩减"
fi
```

若 `PAGE > 1` 则摘要超出，按比例缩减各问题段字数后重新编译，直到 `PAGE=1`。

**禁止**：公式、模型检验、超字数、摘要末尾加 `\newpage`。

|------|------|------|

#### 2. 总体分析

| 部分 | 格式 | 要求 |
|------|------|------|
| 文字 | 三段式 | 第1段：一句话总体表达。第2段：针对问题X建立了XX来解决XX（逐问串联）。第3段：总结各问题之间的递进关系 |

不画流程图。

- 两段式：第1段"为简化问题，本文做出以下假设："

#### 4. 符号说明（全论文表格统一模板）

**所有表格统一使用 `longtable` + `>{\centering\arraybackslash}p{}`，总宽=`\textwidth`，各列居中可跨页，宽度按内容比例分配。**

```latex
\begin{longtable}{>{\centering\arraybackslash}p{0.18\textwidth}>{\centering\arraybackslash}p{0.78\textwidth}}
  \caption{表名（中文）}
  \label{tab:标签} \\
  \toprule
  列1 & 列2 \\
  \midrule
  \endfirsthead
  \caption{表名（续）} \\
  \toprule
  列1 & 列2 \\
  \midrule
  \endhead
  \bottomrule
  \endfoot
  内容 & ... \\
  ... & ...
\end{longtable}
```

**列宽规则（强制）**：

`\tabcolsep` 每列左右各6pt，N列共12N pt，超 `\textwidth` 则右侧溢出。因此各 `p{比例\textwidth}` 比例总和**不可等于1.0**，须按下表预留余量：

| 列数 | 比例总和 | 示例分配（左侧标签列窄→右侧内容列宽） |
|------|----------|----------------------------------------|
| 2列 | **0.96** | `{0.18+0.78}` |
| 3列 | **0.92** | `{0.16+0.16+0.60}` |
| 4列 | **0.88** | `{0.12+0.12+0.12+0.52}` |
| 5列 | **0.84** | `{0.12+0.18+0.18+0.18+0.18}` |
| 6列 | **0.80** | `{0.10+0.14+0.14+0.14+0.14+0.14}` |
| 7列 | **0.76** | `{0.10+0.11+0.11+0.11+0.11+0.11+0.11}` |
| 8列 | **0.72** | `{0.09+0.09+0.09+0.09+0.09+0.09+0.09+0.09}` |
| 9列 | **0.68** | `{0.08+0.075+0.075+0.075+0.075+0.075+0.075+0.075+0.075}` |
| 10列 | **0.64** | `{0.064+0.064+0.064+0.064+0.064+0.064+0.064+0.064+0.064+0.064}` |

通式：比例总和 = 1.04 − 0.04 × N（N为列数）。示例中左侧标签列取窄值，各列可据实际最长内容微调，但**总和必须等于该列数对应的强制值**。

- 所有列居中：`>{\centering\arraybackslash}` 不可省略，否则 `\\` 被干扰导致 `Misplaced \noalign`

**表格规则（强制）**：
- **全论文所有表格（符号说明、算法对比、预处理对比、模型结果、检验分析等一切表格）必须统一使用上述符号说明处定义的 `longtable` + `>{\centering\arraybackslash}p{}` 样式，不得使用 `tabular`、`tabularx` 等其他表格环境**
- `\caption{}` 在 `\label{}` 之前，`\label{}` 后紧跟 `\\`
- `\endfirsthead` / `\endhead` / `\endfoot` 固定结构，跨页时自动打表头
- `\toprule/\midrule/\bottomrule` 固定，`\endfoot` 含 `\bottomrule`，表末不再重复写
- 格子文字一行显示，禁止换行
- 编译后 `grep 'Overfull.*hbox'` 复查

**数据预处理**（仅问题1）：

必须包含以下部分，逐段展开：

| 部分 | 内容 | 说明 |
|------|------|------|
| 第1段 | 预处理原因 | 说明原始数据的来源、格式与存在问题（如缺失值、量纲不统一、异常值等），强调预处理的必要性 |
| 第2段 | 数据理解与提取 | 从附件数据中提取关键信息（订单/距离/时间窗/坐标等），列出提取后的特征数量与样本规模 |
| 第3段 | 预处理步骤 | 逐项列出：数据清洗（缺失值处理、异常值检测）→ 标准化/归一化（如Min-Max、Z-score，写公式）→ 特征提取/衍生（如有） |
| 第4段 | 预处理结果 | 展示处理前后的关键统计量对比（min/max/mean/std），可选对比表或图；说明处理后数据的规模与结构 |
| 第5段 | 总结 | 预处理完成，数据已规整，为后续建模提供清洁输入 |

**预处理公式示例：**

Min-Max归一化：
\begin{equation}
x' = \frac{x - x_{min}}{x_{max} - x_{min}}
\end{equation}

Z-score标准化：
\begin{equation}
x' = \frac{x - \mu}{\sigma}
\end{equation}

其中，$x_{min}$、$x_{max}$分别为最小值和最大值，$\mu$为均值，$\sigma$为标准差。

##### 5.X.2 建模与求解

包含**模型建立**和**模型求解**两部分。两者必须在同一文件——求解大量引用建立的公式/变量/参数，拆开会导致符号不一致。

**模型建立**：每个5.X子文件的核心主体，占最大篇幅。结构为**三段式**：

- **第1段（开篇引文）**：点明"针对问题X，本节将从XX角度建立XX模型"，概述建模思路，"具体过程如下"收尾。
- **第2段（中间展开）**：按编号步骤逐步建模，每步一个 `\textbf{步骤N：标题}`，后跟公式推导段。公式之间用文字串联推导逻辑，不能孤立堆砌。每步包含：问题引入→公式建立→推导过程→变量说明→中间结果。
- **第3段（总结桥接）**："综上所述，本节完成了XX模型的建立，得到了目标函数/评价体系/预测模型，接下来将基于此进行求解。"

**中间展开规则：**

| 要求 | 说明 |
|------|------|
| 公式数量 | 每个编号步骤至少6-7个公式，能推导的每一步都写出公式 |
| 推导详细 | 从基本形式出发，逐步展开到最终形式，中间过程不可跳步 |
| 文字串联 | 公式之间用"将式(X)代入式(Y)可得""由XX性质可知""进一步展开得"等过渡 |
| 变量说明 | 每个公式后紧跟变量说明段，解释每个符号含义 |

**各类型中间步骤划分：**

| 类型 | 编号步骤 |
|------|----------|
| 优化类 | 1.目标函数的建立（2-3公式：基本形式→考虑权重→最终形式）→ 2.约束条件的建立（逐约束编号，2.X 时间约束→2.X 容量约束等，每约束1-2公式）→ 3.核心机理建模（专项子模型，如时变函数/能耗函数/动力学方程，3-5公式）→ 4.求解算法建模（算法流程公式+参数表） |
| 评价类 | 1.评价指标体系的构建（指标表+无量纲化公式）→ 2.指标权重确定（熵权法/层次分析法完整推导，包含信息熵公式→权重公式，展示中间计算结果表）→ 3.综合评价方法（TOPSIS/模糊综合评价，完整公式链：加权矩阵→正负理想解→贴近度，至少5-6公式） |
| 分类/预测类 | 1.特征工程（相关性公式+降维推导+特征表）→ 2.模型选定与构建（模型数学表达式完整写出，包含损失函数/优化目标/正则项）→ 3.模型训练与评估（评估指标公式+训练策略描述） |
| 机理分析 | 1.机理分析框架（核心变量关系图/方程组）→ 2.关键变量关系建模（每个关系写公式+推导）→ 3.模型整合与参数确定（联立方程+参数辨识公式） |

**通用规则（不变）：**
- 每个公式用编号标题引出：`\textbf{标题：}`描述 + `\begin{equation}...\end{equation}` + 变量说明段
- 变量说明段格式：`其中，$X$表示...，$Y$表示...。`（自然段，不分点）
- 公式不单独编号，用 `equation` 环境即可

**注意**：优化类必须包含完整的优化模型三要素（目标函数+决策变量+约束条件），公式必须完整写出不可用文字概括。

**模型求解**：引文→表/图→分析。数据结果用表格突出，趋势/对比用图展示。图内只允许：数据、坐标轴标签刻度、图例（全中文）。不出现分点、加粗。

## 三、代码规范

### 绘图库

只允许 matplotlib，严禁 seaborn 及任何其他可视化库。

| 禁止 | 替代 |
|------|------|
| `sns.set_style()` | `plt.rcParams` + `ax.grid(alpha=0.3, linestyle='--')` |
| `sns.color_palette()` | `plt.cm.viridis(np.linspace(0.1, 0.9, n))` |
| `sns.heatmap()` | `ax.imshow()` + `plt.colorbar()` |
| `sns.despine()` | `ax.spines['top'].set_visible(False)` 等 |

### 两阶段执行（先算后画）

每个求解脚本必须按此顺序：

```
第一阶段：纯计算
  数据加载 → 预处理 → 建模 → 求解 → 得到所有数值结果

第二阶段：检查 + 绘图
   → 打印每组数据的 min/max/mean/std/CV/amplitude（供论文引用）
  → 每种分析结果都画图
```

### 绘图原则

每题至少 4-6 张图（覆盖主要分析维度），多多益善。数据和图的对应关系：

| 图表类型 | 适用场景 |
|----------|----------|
| 柱状图 | 类别对比、分组对比 |
| 折线图 | 趋势变化、时间序列 |
| 热力图 | 矩阵数据、相关性 |
| 散点图 | 两变量关系、分布 |
| 饼图 | 占比构成 |
| 箱线图 | 多组分布对比 |
| 通用 | 主要分析维度必画，多多益善 |

### 图表美观

- 配色：内置 colormap（viridis、RdBu_r、tab10、Set3）
- 折线图：线宽 ≥ 1.5pt，带标记点
- 柱状图：柔和色 + 白边 `edgecolor='white', linewidth=0.5`
- 散点图：`alpha=0.7`, `edgecolors='black', linewidth=0.5`
- 网格：`ax.grid(alpha=0.3, linestyle='--')`
- 去边框：`ax.spines['top'].set_visible(False)` + `right`
- **图上不画 `set_title()`**，标题由论文 `\caption{}` 承担
- 图内只允许：数据、坐标轴标签刻度、图例（全中文）
