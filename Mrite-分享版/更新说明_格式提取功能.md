# Mrite Skill 更新说明：论文格式提取功能

## 📌 更新摘要

在原有的"竞赛题目求解"功能基础上，新增了**论文格式提取模式**。现在你可以用任意一篇参考论文来生成适配所有竞赛题目的新模板，一键替换原有 `论文/` 文件夹，让你的输出论文采用心仪的格式风格。

---

## 🆕 新增功能：论文格式提取模式

### 触发方式

对 Claude 说以下任一触发词即可：
- "提取论文格式"
- "学习这篇论文的写法"
- "按这篇论文做模板"
- "根据这篇论文做模板"

### 输入要求

1. 将参考论文（PDF 或 DOCX 格式）放入 `题目/` 文件夹
2. 可选：放入数据文件到 `数据/` 文件夹（用于理解数据引用格式）

### 输出产物

```
论文_新模板/
├── 论文.tex                      ← 主文件（导言区、封面格式来自参考论文）
├── format.cls                    ← 样式文件（复制自原项目）
├── fonts/                        ← 字体文件夹（复制自原项目）
├── 0.摘要.tex                    ← 10 个固定章节文件（内容风格来自参考论文）
├── 1.引言.tex
├── 2.总体分析.tex
├── 3.模型假设.tex
├── 4.符号说明.tex
├── 5.模型的建立与求解.tex
├── 5.1.问题1的建立求解.tex     ← 第5章子文件（示例）
├── 5.1.1.分析与准备.tex
├── 5.1.2.建模与求解.tex
├── 6.模型检验.tex
├── 7.模型评价.tex
├── 8.模型改进推广.tex
├── 9.参考文献.tex
├── 10.附录.tex
├── README.md                     ← 模板配置说明 + 使用方法 + 示例代码
├── 论文结构分析.md               ← 参考论文的章节结构分析报告
└── 参考论文原文.txt              ← 提取的参考论文全文
```

### 核心设计原则：**替换对齐**

新模板的文件名与原 `论文/` 目录**完全一致**（`0.摘要.tex` ~ `10.附录.tex` 固定不变），这样产物可以**一键替换**原模板，无需修改求解流程代码。

参考论文的格式特征（封面布局、字体设置、宏包选择、表格样式、图片宽度、参考文献格式、章节段落组织模式等）被**注入到固定文件的内容**里，而不是改变文件骨架。

### 使用新模板

```bash
# 1. 备份当前模板
mv 论文 论文_备份

# 2. 启用新模板
mv 论文_新模板 论文

# 3. 开始求解
# 对 Claude 说"开始求解"，后续流程完全不变
```

## 🔧 执行特点

### 全自动模式（与求解模式一致）

- ✅ 不询问、不确认、不等待、不中断
- ✅ 从读取参考论文到生成可编译的模板一气呵成
- ✅ 自动编译测试，0 Error 才算完成
- ✅ 自动修复编译错误（补充宏包、调整语法）

### 格式提取内容

| 提取维度 | 说明 |
|---------|------|
| **章节层次** | 识别一级/二级/三级标题，构建章节树状结构 |
| **封面信息** | 标题位置、作者信息、单位、日期格式、特殊标识 |
| **摘要格式** | 摘要位置、关键词格式、段落组织模式 |
| **正文格式** | 段落缩进、行距、字体大小、页边距 |
| **图表格式** | 图片宽度、标题位置、表格样式、列宽设置方式 |
| **公式格式** | 编号位置、编号格式 |
| **参考文献** | 管理方式（bibtex/纯tex）、引用样式、条目格式 |
| **特殊元素** | 流程图、算法伪代码、符号表、缩略语表 |
| **段落模式** | 每章节的段落数量、功能、过渡词、句式 |

### 智能映射规则

参考论文的章节标题无论叫什么名字，都会被智能映射到 10 个固定目标文件：

| 参考论文可能的章节名 | 映射目标文件 |
|-------------------|-------------|
| 摘要 / Abstract / 内容提要 | `0.摘要.tex` |
| 引言 / 研究背景 / Introduction / 问题重述 | `1.引言.tex` |
| 总体分析 / 问题分析 / 总体思路 | `2.总体分析.tex` |
| 模型假设 / 基本假设 / Assumptions | `3.模型假设.tex` |
| 符号说明 / 变量说明 / 记号约定 | `4.符号说明.tex` |
| 模型建立 / 方法 / 模型与求解 / 主体章节 | `5.模型的建立与求解.tex` 及子文件 |
| 模型检验 / 误差分析 / 灵敏度分析 / 验证 | `6.模型检验.tex` |
| 模型评价 / 优缺点 / 讨论 | `7.模型评价.tex` |
| 模型改进 / 推广 / 展望 | `8.模型改进推广.tex` |
| 参考文献 / References | `9.参考文献.tex` |
| 附录 / 附件 / 代码清单 | `10.附录.tex` |

## 📋 技术细节

### 提取流程（5 步）

1. **Step 0**：读取参考论文（PDF/DOCX），保存原文
2. **Step 1**：结构分析（章节层次、格式特征、段落模式、特殊元素）
3. **Step 2**：生成新模板文件夹（10 个章节文件 + 主文件 + 样式文件）
4. **Step 3**：生成配置说明文档（README.md + 结构分析 + 差异对比）
5. **Step 4**：编译测试（xelatex × 2，自动修复错误直到 0 Error）
6. **Step 5**：输出总结（文件清单 + 使用步骤 + 特征说明）

### 关键规则

- **文件名固定**：10 个章节文件名不变（`0.摘要.tex` ~ `10.附录.tex`）
- **风格注入**：参考论文的格式特征注入到固定文件的**内容**
- **路径统一**：所有图片路径改写为 `../求解/问题X/图片/xxx.png`
- **辅助文档落位**：分析报告、原文、说明全部放在 `论文_新模板/` 内部
